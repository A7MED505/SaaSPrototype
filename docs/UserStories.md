# User Stories

## User Story 1: User Registration
**As a** new user
**I want to** create an account with email and password
**So that** I can access the e-commerce platform

### Acceptance Criteria
- [ ] User can navigate to registration page
- [ ] User can enter email, password, and name
- [ ] System validates email format
- [ ] System validates password strength
- [ ] User receives confirmation message
- [ ] User is redirected to login page

### Test Cases
1. Register with valid data
2. Register with invalid email
3. Register with weak password
4. Register with existing email

---

## User Story 2: Product Browsing
**As a** customer
**I want to** view all available products
**So that** I can find items to purchase

### Acceptance Criteria
- [ ] User can see product list on homepage
- [ ] Each product shows name, price, image
- [ ] User can filter by category
- [ ] User can search for products
- [ ] User can sort by price/rating
- [ ] User can view product details

### Test Cases
1. Load product list
2. Filter by category
3. Search for product
4. Sort by price
5. View product details

---

## User Story 3: Shopping Cart
**As a** customer
**I want to** add items to my cart
**So that** I can purchase multiple items

### Acceptance Criteria
- [ ] User can add items to cart
- [ ] Cart count updates
- [ ] User can view cart
- [ ] User can remove items
- [ ] User can see total price
- [ ] Cart persists after logout

### Test Cases
1. Add item to cart
2. Remove item from cart
3. Update quantity
4. View cart total
5. Cart persistence

---

## User Story 4: Checkout
**As a** customer
**I want to** complete my purchase
**So that** I can receive my items

### Acceptance Criteria
- [ ] User can proceed to checkout
- [ ] User can enter shipping address
- [ ] User can select shipping method
- [ ] User can enter payment info
- [ ] User receives order confirmation
- [ ] Order is saved to database

### Test Cases
1. Checkout with valid address
2. Checkout with invalid address
3. Select shipping method
4. Process payment
5. Receive confirmation

---

## User Story 5: Admin Product Management
**As an** admin

**So that** I can keep inventory updated

### Acceptance Criteria
- [ ] Admin can add new products
- [ ] Admin can edit products
- [ ] Admin can delete products
- [ ] Admin can view inventory
- [ ] Admin can set prices
- [ ] Changes are reflected immediately

### Test Cases
1. Add new product
2. Edit product details
3. Delete product
4. Update inventory
5. View product list

## User Story 6: User Profile
**As a** customer

**So that** I can keep my information updated

### Acceptance Criteria

- [ ] User can edit name and email
